# مكتبة التيمبليتس — شرح اللوجيك للتيم

> هذا الملف مخصص لشرح فكرة مكتبة التيمبليتس بطريقة مبسطة — بدون كود.

---

## الفكرة ببساطة

مكتبة التيمبليتس = **مكان فيه قوالب جاهزة** (Campaign Brief, Content Plan, Media Plan...).  
أي حد في الشركة يقدر ياخد **نسخة** من أي قالب، يملأها ببياناته، ويحفظها عنده.  
المستند اللي يعمله بيتحفظ في **مدير الملفات** ويترفع على **Google Drive** تلقائي.

---

## مين يعمل إيه؟

| الدور | يقدر يعمل إيه |
|------|--------------|
| **كل الموظفين** | يشوفوا المكتبة — يعاينوا أي قالب — ياخدوا نسخة ويملوها — يحفظوا مستنداتهم في الدرايف |
| **المدير / الأدمن** | كل اللي فوق + إنشاء قوالب جديدة + تعديلها + حذفها + نسخها |

> ⚠️ **قاعدة مهمة**: الموظف العادي مش بيعدّل على القالب الأصلي أبداً — بياخد نسخة بس. القالب الأصلي **مقفول** (read-only).

## رحلة المستخدم — خطوة بخطوة

### المسار الأساسي: من المكتبة للحفظ

```
📚 يفتح مكتبة التيمبليتس
        │
        ▼
🔍 يبحث أو يفلتر بالقسم (Marketing / Sales / Content / Dev)
        │
        ▼
👁️ يضغط "معاينة" على القالب اللي عايزه
        │
        ▼
   يشوف شكل الحقول والأقسام (بدون ما يعدل)
        │
        ▼
✅ يضغط "استخدم التيمبليت"
        │
        ▼
   📝 النظام يعمل نسخة جديدة (مستند) باسمه
   البيانات فاضية (أو فيها قيم افتراضية)
        │
        ▼
   يفتح صفحة المحرر تلقائي
        │
        ▼
✏️ يملأ البيانات في الفورم (كل قسم بحقوله)
        │
        ▼
   عنده 3 خيارات:
   ┌────────────────────────────────────────────┐
   │  💾 حفظ مسودة      → يحفظ في النظام بس    │
   │                       (يرجعله بعدين يكمل)  │
   │                                            │
   │  📁 حفظ في الدرايف  → يحفظ + يعمل ملف      │
   │                       في مدير الملفات +     │
   │                       يترفع على Drive       │
   │                                            │
   │  ✅ إتمام           → يغيّر الحالة لمكتمل   │
   └────────────────────────────────────────────┘
```

### صفحة "مستنداتي"

كل يوزر عنده صفحة **مستنداتي** فيها كل المستندات اللي عملها:
- يقدر يفلتر بالحالة: **مسودة** / **مكتمل** / **مؤرشف**
- يقدر يدخل أي مستند يعدله أو يكمله
- يقدر يحذف أي مستند بتاعه

## القوالب الجاهزة (7 قوالب)

النظام فيه 7 قوالب جاهزة مقسّمة على 4 أقسام:

### 🟠 Marketing
| القالب | الحقول | الوصف |
|--------|--------|------|
| **Campaign Brief** | 21 حقل، 6 أقسام | أهم قالب — بريف الحملة الإعلانية الكامل |
| **Media Plan** | 10 حقل، 3 أقسام | خطة الميديا والمنصات |

### 🔵 Sales
| القالب | الحقول | الوصف |
|--------|--------|------|
| **Sales Funnel** | 6 حقول | قمع المبيعات |
| **Client Onboarding** | 5 حقول | تأهيل العميل الجديد |

### 🟢 Content
| القالب | الحقول | الوصف |
|--------|--------|------|
| **Content Plan — شهري** | 7 حقول | خطة المحتوى الشهرية |

### 🟣 Development
| القالب | الحقول | الوصف |

## مثال عملي: Campaign Brief

لما الموظف يستخدم قالب Campaign Brief، بيلاقي فورم فيه 6 أقسام:

### القسم 1: معلومات أساسية
| الحقل | النوع |
|------|------|
| اسم الحملة | نص قصير |
| اسم العميل | نص قصير |
| المسؤول عن الحملة | نص قصير |
| تاريخ البداية | تاريخ |
| تاريخ النهاية | تاريخ |

### القسم 2: الهدف والجمهور
| الحقل | النوع |
|------|------|
| هدف الحملة الرئيسي | اختيار (Brand Awareness / Lead Gen / Conversions / Traffic / Engagement / App Installs) |
| الجمهور المستهدف | نص طويل |
| Unique Selling Point | نص طويل |

### القسم 3: المنصات والميزانية
| الحقل | النوع |
|------|------|
| المنصات الإعلانية | اختيار متعدد (Meta Ads / Instagram / TikTok / Google Ads / YouTube / Snapchat) |
| إجمالي الميزانية | مبلغ مالي (EGP) |
| توزيع الميزانية | اختيار (Meta 70%/Google 30% — Meta 60%/TikTok 40% — بالتساوي — Custom) |

### القسم 4: الـ KPIs
| الحقل | النوع |
|------|------|
| عدد الليدز المستهدف | رقم (ليد) |
| أقصى CPL مقبول | مبلغ مالي (EGP) |

### القسم 5: الكريتيف والمحتوى
| الحقل | النوع |
|------|------|
| أنواع الكريتيف | اختيار متعدد (فيديو 15 ث / كاروسيل / Static / Stories / Reels / UGC) |
| الرسالة الرئيسية | نص طويل |
| لهجة المحتوى | اختيار (ملهم / ودود / احترافي / استعجالي) |

### القسم 6: ملاحظات
| الحقل | النوع |
|------|------|
| ملاحظات إضافية | نص طويل |

## كيف بيشتغل "حفظ في الدرايف"؟

```
يضغط "حفظ في الدرايف"
        │
        ▼
النظام بيعمل 4 حاجات:
        │
   ┌────┴────────────────────────────────────────────────┐
   │ 1. يحفظ البيانات في الداتابيز (جدول user_documents) │
   │ 2. يولّد ملف JSON بالبيانات                          │
   │ 3. يعمل record في مدير الملفات (managed_files)       │
   │    في فولدر اسمه "مستندات التيمبليتس"                │
   │ 4. يرفع الملف على Google Drive تلقائي               │
   │    (لو الشركة متوصلة بـ Drive)                       │
   └─────────────────────────────────────────────────────┘
        │
        ▼
المستند حالته بتتغير من "مسودة" → "مكتمل"
        │
        ▼
الملف بيظهر في:
  📁 مدير الملفات → فولدر "مستندات التيمبليتس"
  ☁️ Google Drive → نفس الفولدر متسنك
```

لما أي حد يحفظ مستند في الدرايف، بيروح هنا:

```
📁 مستندات التيمبليتس/
   📄 Campaign Brief — 2026-04-04 14-30.json
   📄 Content Plan — شهري — 2026-04-04 15-00.json
   📄 Media Plan — 2026-04-05 10-00.json
```

> الفولدر ده بيتعمل أوتوماتيك أول مرة حد يحفظ مستند. مفيش فولدرات فرعية حاليًا.

## حماية القوالب — 3 مبادئ

### 1. القالب الأصلي مقفول 🔒
- الموظف العادي مش بيقدر يعدل أو يحذف القالب
- بس الأدمن والمدير يقدروا
- لما الموظف يضغط "استخدم"، بياخد **نسخة** — مش بيعدل الأصل

### 2. النسخة مستقلة تماماً 📸
- لما الموظف ياخد نسخة، النظام بيحفظ **snapshot كامل** من هيكل القالب
- لو الأدمن عدّل القالب الأصلي بعدين أو حذفه → مستندات الموظفين **مش بتتأثر**
- كل مستند شايل نسخته الخاصة من الهيكل

### 3. كل واحد يشوف بتاعه بس 👤
- كل موظف يشوف المستندات اللي هو عملها
- المدير يقدر يشوف كل مستندات الشركة
- محدش يقدر يعدل مستند غيره

## الفرق بين "قوالب الملفات" القديمة و"مكتبة التيمبليتس" الجديدة

| | قوالب الملفات (القديمة) | مكتبة التيمبليتس (الجديدة) |
|---|---|---|
| **النوع** | ملفات فعلية (PDF, DOCX) | قوالب ذكية بحقول |
| **الاستخدام** | تحميل الملف وتعديله برا النظام | ملء الحقول جوا النظام مباشرة |
| **المعاينة** | مفيش | ✅ معاينة الحقول والأقسام |
| **النسخ** | مفيش | ✅ نسخة مستقلة لكل يوزر |
| **الحفظ** | مرفوع كملف بس | ✅ بيانات + ملف في File Manager + Drive |
| **المحرر** | مفيش | ✅ فورم ديناميكي جوا النظام |

## الأسئلة المتوقعة

### "هل لو حذفت القالب المستندات هتتحذف؟"
**لا.** المستندات مستقلة تماماً. لو القالب اتحذف، المستندات اللي اليوزرز عملوها تفضل شغالة بالهيكل اللي كان موجود وقت الإنشاء.

### "هل أقدر أعمل قالب جديد؟"
**أيوا** — المدير والأدمن يقدروا يعملوا قوالب جديدة ويحددوا الحقول والأقسام. أو ينسخوا قالب موجود ويعدلوه.

### "هل المستند بيروح Drive؟"
**اختياري.** اليوزر يقدر يحفظ مسودة بس (في النظام) أو يحفظ في الدرايف (وده بيعمل ملف في مدير الملفات + Drive).

### "هل أي حد يقدر يشوف مستنداتي؟"
**لا.** كل يوزر يشوف بتاعه بس. المدير يقدر يشوف مستندات الشركة كلها.

### "هل أي حد يقدر يعدل على القوالب؟"
**لا.** بس الأدمن والمدير. الموظف العادي بياخد نسخة — مش بيعدل الأصل.

### "هل القوالب دي لكل الشركات ولا لكل شركة لوحدها؟"
**لكل شركة لوحدها.** كل شركة عندها أقسامها وقوالبها الخاصة. النظام بيعمل نفس الـ 7 قوالب لكل شركة أوتوماتيك.
