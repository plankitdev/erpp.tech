# دليل رحلة الموظف في نظام ERPFlex

> آخر تحديث: أبريل 2026  
> يشرح هذا الملف تجربة كل دور في النظام بالتفصيل — من أول تسجيل دخول حتى إنهاء العمل اليومي.

---

## أولاً: هيكل الأدوار

```
super_admin
    └── company_admin  (مدير الشركة)
            ├── manager          (مدير تنفيذي)
            ├── marketing_manager  (أكونت مانجر / مدير التسويق)
            ├── accountant       (محاسب)
            ├── sales            (مبيعات)
            └── employee         (موظف)
```

---

## ثانياً: رحلة كل دور

---

### 👤 الموظف العادي (`employee`)

**ما يشوفه في الـ Sidebar:**
- الإعلانات / المحادثات (متاحة للكل)
- مهامي الشخصية
- المهام
- المشاريع
- التقويم / الاجتماعات
- تتبع الوقت
- لوحة الأداء (KPI)
- الموارد البشرية: حضوري، إجازاتي، رواتبي

**رحلته اليومية:**
1. **تسجيل الدخول** ← يشوف الـ Dashboard بالمهام المعينة له والإشعارات
2. **المهام** ← يشوف مهامه هو فقط (مش مهام التيم كله)
   - يغير الحالة: `todo → in_progress → review`
   - **لا يقدر يضغط "منجزة" بنفسه** — لازم الأكونت مانجر يوافق
3. **يسلّم المهمة:** يضغط "إرسال للمراجعة" ← تاخد status = `review`
4. **ينتظر الموافقة:** لو الأكونت مانجر وافق، يوصله إشعار تلقائي ✅
5. **لو رُفضت:** يوصله إشعار مع سبب الرفض — يرجع يعدل ويبعت تاني
6. **مهامي الشخصية** ← تو-دو ليست خاصة بيه مش مرتبطة بمشاريع
7. **تسجيل الحضور:** من صفحة الحضور يسجل دخول/خروج يومي
8. **طلب إجازة:** من صفحة الإجازات يقدم طلب ← الـ HR/Admin يوافق
9. **الشات:** يتكلم مع أي عضو في الشركة أو في قنوات جماعية
10. **KPI:** يشوف أداءه الخاص فقط — التارجت والإنجاز الفعلي

**ما لا يستطيع فعله:**
- ❌ رؤية مهام زملاؤه
- ❌ الموافقة على أي مهمة
- ❌ رؤية الفواتير أو الخزينة
- ❌ إنشاء مشاريع
- ❌ رؤية رواتب الآخرين

### 📣 الأكونت مانجر (`marketing_manager`)

> **هذا هو الدور المحوري في إدارة العمل اليومي للتيم**

**ما يشوفه في الـ Sidebar:**
- كل ما يشوفه الموظف +
- **لوحة الأكونت مانجر** ⭐
- **التقرير الأسبوعي** ⭐
- **تقرير العميل** ⭐
- المشاريع (عرض + إنشاء + تعديل + حذف)
- العملاء (عرض فقط)
- الموظفين (عرض فقط)
- التقارير (عرض فقط)

#### لوحة الأكونت مانجر (`/account-manager`)

اللوحة الرئيسية تتضمن 6 أقسام:

**1. الإحصائيات السريعة (Stats Bar)**
```
المشاريع النشطة | المهام المفتوحة | متأخرة | تنتظر مراجعة | منجزة اليوم | عدد التيم
```

**2. المهام العاجلة اليوم**
- المهام المستحقة اليوم أو المتأخرة (بحد أقصى 15)
- مرتبة: عالية الأولوية أولاً ثم بالتاريخ
- تظهر: اسم المهمة، المُعيَّن إليه، المشروع، هل متأخرة

**3. المشاريع النشطة**
- كل مشروع active مع: نسبة التقدم، أيام متبقية، عدد المهام المتأخرة، في مرحلة مراجعة
- لو `days_left < 0` → يظهر بلون أحمر (متأخر عن الموعد)

**4. تحميل الشغل على التيم (Team Workload)**
- كل عضو في الشركة مع: إجمالي مهامه المفتوحة، متأخرة، في مراجعة، قيد التنفيذ، todo

**5. تنتظر المراجعة (Pending Review)** ⭐
أهم قسم في اللوحة — المهام اللي بعتها الموظفين للمراجعة:
- **زرار "موافقة"** ← تنقل المهمة إلى `done` + يوصل إشعار للموظف
- **زرار "رفض"** ← تفتح modal لكتابة سبب الرفض + يوصل إشعار للموظف
- **زرار "💬 تعليق"** ← يفتح input inline بدون فتح صفحة جديدة، اكتب + Enter

**6. المهام المتكررة**
- المهام اللي ليها recurrence (يومي/أسبوعي/شهري) وما اتعملتش

#### التقرير الأسبوعي (`/weekly-report`)

```
الأسبوع الحالي: من [تاريخ] إلى [تاريخ]
┌─────────────┬──────────────┬─────────────────┐
│ منجزة      │ منشأة        │ الأفضل أداءً    │
│ (أخضر)    │ (أزرق)       │ (ذهبي)          │
└─────────────┴──────────────┴─────────────────┘
شريط نسبة الإنجاز: XX%

قائمة الأعضاء مرتبة بالأداء:
#1 [اسم] — منجزة: X | منشأة: X | متأخرة: X
   ▸ [توسيع] → قائمة المهام المنجزة + المتأخرة
```

#### تقرير العميل (`/client-report`)

```
[اختر عميل ▼]

بعد الاختيار:
┌──────┬──────────────┬──────────┬─────────┐
│ todo │ قيد التنفيذ  │ مراجعة  │ منجزة  │
└──────┴──────────────┴──────────┴─────────┘
نسبة الإنجاز الكلية: XX%

مشروع 1: [اسم] ──────────── 65%  [رابط للمشروع]
مشروع 2: [اسم] ──────────── 20%  [رابط للمشروع]
```

**رحلة الأكونت مانجر اليومية:**
1. يدخل **لوحة الأكونت مانجر** ← يشوف الصورة الكاملة
2. يراجع **المهام العاجلة** ← يعيّن اللي محتاج تعيين أو يتابع المتأخرة
3. يعالج **تنتظر المراجعة** ← يوافق / يرفض / يعلق
4. يراجع **تحميل التيم** ← يوزع الشغل لو حد مثقّل أو فاضي
5. آخر الأسبوع ← يفتح **التقرير الأسبوعي** ← يعرف مين شغّال ومين لأ
6. قبل اجتماع عميل ← يفتح **تقرير العميل** ← جاهز بأرقام حقيقية

### 🛒 موظف المبيعات (`sales`)

**ما يشوفه:**
- العملاء (عرض + إنشاء + تعديل — مش حذف)
- العقود (عرض + إنشاء)
- الفواتير (عرض + إنشاء)
- عروض الأسعار
- العملاء المحتملين (Leads)
- تذاكر الدعم
- البريد الإلكتروني
- المهام (عرض + إنشاء + تعديل)

**رحلته:**
1. **Leads:** يُدخل عميل محتمل جديد من السوق
2. **يحوله لعميل** حقيقي بعد الاهتمام
3. **يعمل عرض سعر** → يبعته للعميل → لو قبل، يحوله لعقد
4. **يعمل فاتورة** مرتبطة بالعقد
5. **تذاكر الدعم:** يستقبل شكاوى العملاء ويتابعها

---

### 💰 المحاسب (`accountant`)

**ما يشوفه:**
- الفواتير (كامل الصلاحيات)
- الخزينة (كامل الصلاحيات)
- المصروفات (كامل الصلاحيات)
- المرتبات (كامل الصلاحيات)
- العملاء (عرض فقط)
- الموظفين (عرض فقط)
- المهام (عرض + إنشاء + تعديل)
- التقارير المالية (عرض فقط)

**رحلته:**
1. يراجع الفواتير المعلقة ← يسجل المدفوعات
2. يدخل المصروفات اليومية
3. يراجع حركة الخزينة
4. آخر الشهر: يصرف المرتبات
5. يراجع التقارير المالية — الربح والخسارة، مؤشرات الأداء

---

### 📊 المدير التنفيذي (`manager`)

**صلاحيات أوسع من الأكونت مانجر:**
- كل ما يشوفه الأكونت مانجر +
- **الفواتير** (كامل الصلاحيات)
- **العقود** (كامل الصلاحيات)
- **الخزينة** (عرض + إنشاء)
- **المرتبات** (عرض + إنشاء)
- **الموظفين** (كامل الصلاحيات)
- **التقارير** (عرض)
- **اليوزرز** (عرض + إنشاء)

---

### 🏢 مدير الشركة (`company_admin`)

**كل الصلاحيات** داخل شركته بدون استثناء:
- ينشئ ويحذف يوزرز
- يعدل الصلاحيات بشكل مخصص لكل موظف
- يشوف كل التقارير
- يعدل إعدادات الشركة

---

### 👑 السوبر أدمن (`super_admin`)

- **يشوف كل الشركات**
- يقدر يعمل Switch لأي شركة من الـ token
- الوحيد اللي يقدر يحذف يوزرز
- ما محتاجش company_id

## ثالثاً: دورة حياة المهمة الكاملة

```
[إنشاء المهمة]
      │
      ▼ (assigned_to = الموظف)
  ┌─────────┐
  │  todo   │  ← الموظف يشوفها في قائمة مهامه
  └────┬────┘
       │ يبدأ الشغل
       ▼
  ┌─────────────┐
  │ in_progress │  ← الأكونت مانجر يشوفها في "تحميل التيم"
  └──────┬──────┘
         │ الموظف يضغط "إرسال للمراجعة"
         ▼
  ┌────────────┐
  │   review   │  ← تظهر في "تنتظر المراجعة" في AM Hub
  └──────┬─────┘
         │
    ┌────┴────┐
    │         │
    ▼         ▼
 موافقة      رفض
    │         │
    ▼         ▼
  ┌──────┐  ┌──────────────────────────┐
  │ done │  │ رجوع لـ in_progress      │
  └──────┘  │ + إشعار مع سبب الرفض   │
  + إشعار  └──────────────────────────┘
  للموظف        ← الموظف يعدل ويبعت تاني
```

## رابعاً: نظام الإشعارات

| الحدث | من يستقبل الإشعار |
|---|---|
| تعيين مهمة جديدة | الموظف المُعيَّن |
| اقتراب موعد المهمة | الموظف المُعيَّن |
| مهمة تأخرت عن موعدها | الموظف + المدير |
| مهمة أُرسلت للمراجعة | الأكونت مانجر / المدير |
| مهمة تمت الموافقة عليها | الموظف المُعيَّن ✅ |
| مهمة رُفضت مع السبب | الموظف المُعيَّن ❌ |
| تعليق جديد على مهمة | المُعيَّن + المنشئ |
