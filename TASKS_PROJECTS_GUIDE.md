# دليل المهام والمشاريع - ERPFlex
## User Journey الكامل

---

## 📌 نظرة عامة

نظام المهام والمشاريع في ERPFlex يتيح إدارة كاملة لدورة حياة المشاريع من الإنشاء حتى التسليم، مع تتبع المهام، تسجيل الوقت، قوائم المراجعة، والتعليقات.

---

## 🔐 الصلاحيات حسب الدور

| الإجراء | Super Admin | Manager | Employee |
|---------|:-----------:|:-------:|:--------:|
| رؤية كل المشاريع | ✅ | ✅ | ✅ |
| إنشاء مشروع | ✅ | ✅ | ❌ |
| تعديل/حذف مشروع | ✅ | ✅ | ❌ |
| رؤية كل المهام | ✅ | ✅ | ❌ |
| رؤية المهام المنسوبة فقط | — | — | ✅ |
| إنشاء مهمة | ✅ | ✅ | ✅ |
| تعديل مهمة | ✅ | ✅ | ✅ (المنسوبة له فقط) |
| حذف مهمة | ✅ | ✅ | ❌ |
| حذف مهام متعددة (Batch) | ✅ | ✅ | ❌ |
| رفع ملفات للمشروع | ✅ | ✅ | ❌ |
| إدارة قوائم المراجعة | ✅ | ✅ | ✅ |
| إضافة تعليقات | ✅ | ✅ | ✅ |
| تسجيل الوقت (Timer) | ✅ | ✅ | ✅ |

### ملاحظة مهمة للموظفين:
- **المهام**: الموظف يرى فقط المهام المنسوبة إليه (سواء كمسؤول رئيسي أو ضمن المسؤولين المتعددين)
- **المشاريع**: الموظف يرى كل مشاريع الشركة لكن لا يستطيع إنشاء/تعديل/حذف مشروع

## 🗂️ المشاريع (Projects)

### رحلة إنشاء مشروع (Manager / Super Admin)

#### 1. الدخول لصفحة المشاريع
**المسار**: `/projects`

عند الدخول تظهر:
- **شريط بحث** للبحث بالاسم
- **فلاتر الحالة**: الكل | نشط | مكتمل | متوقف
- **زر "مشروع جديد"**
- **عرض شبكي أو جدولي** (Grid / Table toggle)

#### 2. اختيار القالب
عند الضغط على "مشروع جديد" تظهر **13 قالب جاهز**:

| القالب | الوصف |
|--------|-------|
| 🏢 مشروع فارغ | بدون مهام مسبقة |
| 📝 خطة محتوى | مهام لإنشاء وإدارة محتوى |
| 📱 سوشيال ميديا | حملة تسويق عبر وسائل التواصل |
| 📢 حملة تسويقية | خطوات تنفيذ حملة تسويقية |
| 🎨 هوية بصرية | تصميم هوية العلامة التجارية |
| 🌐 موقع إلكتروني | تطوير موقع ويب |
| 🛒 متجر إلكتروني | بناء متجر أونلاين |
| 📱 تطبيق جوال | تطوير تطبيق |
| 💻 مشروع برمجي | تطوير برمجيات |
| 🎬 إنتاج فيديو | إنتاج محتوى مرئي |
| 🔍 SEO | تحسين محركات البحث |
| 🎨 UI/UX | تصميم واجهات المستخدم |
| 📊 خطة استراتيجية | تخطيط استراتيجي |

كل قالب يحتوي مهام جاهزة تُنشأ تلقائيًا مع المشروع.

#### 3. ملء بيانات المشروع
- **اسم المشروع** (مطلوب)
- **الوصف** (اختياري)
- **العميل** (اختياري - من قائمة العملاء)
- **تاريخ البداية والنهاية** (اختياري)
- **الميزانية** (اختياري)
- **العملة** (EGP / USD / SAR)

#### 4. ما يحدث بعد الإنشاء
- يُنشأ المشروع بحالة **"نشط" (active)**
- تُنشأ **قناة محادثات** تلقائيًا باسم العميل/المشروع
- يُضاف كل موظفي الشركة لقناة المحادثات
- يُرسل **إشعار** لجميع المستخدمين بإنشاء المشروع
- إذا تم اختيار قالب، تُنشأ المهام الجاهزة تلقائيًا

---

### تفاصيل المشروع (Project Detail)
**المسار**: `/projects/:slug`

#### التبويبات:

##### 📋 تبويب المهام (Tasks)
- عرض كل مهام المشروع
- فلترة حسب الحالة (الكل، جديد، جاري، مراجعة، مكتمل)
- إنشاء مهمة جديدة مباشرة من المشروع
- إنشاء مهام فرعية (Subtasks)
- تحديث حالة المهمة بسرعة
- حذف المهام

##### 📁 تبويب الملفات (Files)
- رفع ملفات (حتى 20MB)
- تسمية الملفات
- تحميل/حذف الملفات
- عرض نوع وحجم كل ملف

##### ℹ️ تبويب المعلومات (Info)
- بيانات المشروع (الاسم، الوصف، العميل، التواريخ)
- تعديل البيانات

#### بطاقة البيانات العلوية (Hero Card)
- **حلقة التقدم** (Progress Ring) بنسبة الإنجاز
- **إحصائيات سريعة**: إجمالي المهام، المتبقية، الوقت المسجل، الملفات
- **شريط توزيع الحالات**: يعرض نسبة كل حالة بلون مميز

#### حالات المشروع
| الحالة | اللون | الوصف |
|--------|-------|-------|
| 🟢 نشط (active) | أخضر | المشروع جاري العمل عليه |
| ✅ مكتمل (completed) | أزرق | تم الانتهاء من المشروع |
| ⏸️ متوقف (on_hold) | أصفر | المشروع متوقف مؤقتًا |
| ❌ ملغي (cancelled) | أحمر | تم إلغاء المشروع |

## ✅ المهام (Tasks)

### طرق عرض المهام

#### 1. عرض القائمة (Tasks List)
**المسار**: `/tasks`

- **جدول كامل** بكل المهام مع أعمدة: العنوان، الحالة، الأولوية، المسؤول، العميل، تاريخ الاستحقاق
- **بحث** بالعنوان أو الوصف أو اسم المسؤول
- **فلاتر**: الحالة، الأولوية
- **إحصائيات** في الأعلى: إجمالي، جديد، جاري، مراجعة، مكتمل، متأخر
- **حذف متعدد** (للمدير وsuper admin فقط)
- **تحديث الحالة** مباشرة من القائمة

#### 2. لوحة Kanban (Task Board)
**المسار**: `/tasks/board`

لوحة تفاعلية بنظام السحب والإفلات (Drag & Drop):

| العمود | اللون | الوصف |
|--------|-------|-------|
| 📋 جديد (todo) | رمادي | مهام لم تبدأ بعد |
| 🔄 جاري التنفيذ (in_progress) | أزرق | مهام يتم العمل عليها حاليًا |
| 🔍 مراجعة (review) | بنفسجي | مهام تحتاج مراجعة |
| ✅ مكتمل (done) | أخضر | مهام منتهية |

**ميزات لوحة Kanban**:
- **سحب وإفلات** المهمة من عمود لآخر لتحديث الحالة تلقائيًا
- **فلتر بالمشروع** لعرض مهام مشروع محدد
- كل بطاقة تعرض: العنوان، الأولوية، تقدم القائمة، الوقت المسجل، المسؤول
- المهام مرتبة حسب تاريخ الاستحقاق

#### 3. نظرة عامة (Tasks Hub)
**المسار**: `/tasks-hub`

لوحة إحصائية تعرض:
- **رسم بياني** لتوزيع حالات المهام
- **المشاريع النشطة** مع أشرطة التقدم
- **أهم المهام المتأخرة** (أول 5)
- **الاجتماعات القادمة** (أول 5)
- **معدل الإنجاز**
- **أزرار إجراءات سريعة**

---

### رحلة إنشاء مهمة

#### 1. فتح نموذج الإنشاء
يمكن إنشاء مهمة من:
- صفحة المهام (`/tasks`) → زر "مهمة جديدة"
- لوحة Kanban (`/tasks/board`)
- داخل مشروع (`/projects/:slug`) → تبويب المهام

#### 2. ملء بيانات المهمة
| الحقل | نوع | مطلوب | وصف |
|-------|-----|-------|-----|
| العنوان | نص | ✅ | عنوان المهمة |
| الوصف | نص طويل | ❌ | تفاصيل المهمة |
| الحالة | قائمة | ✅ | جديد / جاري / مراجعة / مكتمل |
| الأولوية | قائمة | ✅ | منخفضة / متوسطة / عالية |
| المسؤول الرئيسي | قائمة مستخدمين | ❌ | الشخص المسؤول عن المهمة |
| المسؤولون الإضافيون | اختيار متعدد | ❌ | مسؤولون إضافيون |
| تاريخ البداية | تاريخ | ❌ | متى تبدأ المهمة |
| تاريخ الاستحقاق | تاريخ | ❌ | الموعد النهائي |
| المشروع | قائمة | ❌ | ربط المهمة بمشروع |
| العميل | قائمة | ❌ | ربط المهمة بعميل |
| التكرار | قائمة | ❌ | بدون / يومي / أسبوعي / شهري |
| مهمة أب | رقم | ❌ | لإنشاء مهمة فرعية |

#### 3. ما يحدث بعد الإنشاء
- يُرسل **إشعار** للمسؤول والمسؤولين الإضافيين (إذا كانوا مختلفين عن المنشئ)
- تظهر المهمة فورًا في كل طرق العرض

---

### تفاصيل المهمة (Task Detail)
**المسار**: `/tasks/:id`

صفحة شاملة تحتوي على:

#### 📝 المعلومات الأساسية
- العنوان (قابل للتعديل)
- الوصف (قابل للتعديل)
- الحالة (4 خيارات مع أيقونات ملونة)
- الأولوية (3 مستويات)
- المسؤول الرئيسي
- المسؤولون الإضافيون
- المشروع المرتبط
- العميل المرتبط
- تواريخ البداية والاستحقاق
- منشئ المهمة وتاريخ الإنشاء

#### ⏱️ تتبع الوقت (Time Tracking)
**كيف يعمل**:
1. اضغط **زر "ابدأ"** لبدء تسجيل الوقت
2. يظهر **عداد مباشر** يحسب الثواني والدقائق
3. يتم استطلاع التايمر كل **30 ثانية** من السيرفر
4. اضغط **"إيقاف"** لحفظ الوقت
5. يمكن إضافة وقت يدويًا (تاريخ بداية ونهاية ومدة)

**البيانات المعروضة**:
- إجمالي الوقت المسجل للمهمة
- قائمة بكل المُدخلات (التاريخ، المدة)
- يمكن حذف مُدخلات الوقت

#### ✔️ قوائم المراجعة (Checklists)
**كيف تعمل**:
1. اكتب عنصر جديد واضغط Enter أو زر الإضافة
2. يظهر العنصر مع **مربع اختيار** (Checkbox)
3. اضغط على المربع لتأشيره كـ **مكتمل**
4. يمكن **حذف** أي عنصر
5. يمكن **إعادة ترتيب** العناصر بالسحب
6. يظهر **شريط تقدم** يعرض نسبة الإنجاز (مثال: 3/5 = 60%)

#### 💬 التعليقات (Comments)
**كيف تعمل**:
1. اكتب تعليقك في مربع النص
2. يمكن إرفاق **ملف** (حتى 10MB)
3. اضغط إرسال
4. يظهر التعليق مع اسم الكاتب والتاريخ
5. المرفقات قابلة للتحميل

#### 🌳 المهام الفرعية (Subtasks)
- يمكن إنشاء **مهام فرعية** داخل مهمة أب
- المهام الفرعية لها نفس خصائص المهمة العادية
- تظهر في قائمة منفصلة داخل تفاصيل المهمة الأب
- كل مهمة فرعية لها مسؤول وحالة خاصة

---

### حالات المهمة وتدفق العمل

```
📋 جديد (todo)
    ↓ [يبدأ العمل]
🔄 جاري التنفيذ (in_progress)
    ↓ [ينتهي من العمل]
🔍 مراجعة (review)
    ↓ [تتم الموافقة]
✅ مكتمل (done)
```

**عند إتمام المهمة (الحالة → مكتمل)**:
- يُرسل **إشعار** لمنشئ المهمة (إذا كان مختلفًا عن من أنهاها)
- يتم تفعيل **أتمتة Workflows** (إذا كان هناك قاعدة مرتبطة)

### مستويات الأولوية

| المستوى | اللون | الوصف |
|---------|-------|-------|
| 🟢 منخفضة (low) | أخضر | مهمة يمكن تأجيلها |
| 🟡 متوسطة (medium) | أصفر | أولوية عادية |
| 🔴 عالية (high) | أحمر | مهمة عاجلة يجب إنجازها أولًا |

### المهام المتكررة (Recurring Tasks)

| النوع | الوصف |
|-------|-------|
| بدون تكرار | مهمة واحدة فقط |
| يومي | تتكرر كل يوم |
| أسبوعي | تتكرر كل أسبوع |
| شهري | تتكرر كل شهر |

عند تفعيل التكرار، يتم حساب `next_recurrence_date` تلقائيًا.

## 📊 الإشعارات والأتمتة

### إشعارات تلقائية

| الحدث | من يستلم الإشعار |
|-------|-------------------|
| تم تكليفك بمهمة | المسؤول عن المهمة + المسؤولون الإضافيون |
| تم إكمال مهمة | منشئ المهمة |
| تم إنشاء مشروع | كل مستخدمي الشركة |

### أتمتة Workflows
عند إكمال مهمة (`status → done`)، يتم تفعيل الـ Workflow Rules المرتبطة بحدث `task_completed`.

## 🔗 الترابط بين الأنظمة

```
المشروع ←→ العميل (client_id)
المشروع ←→ المهام (hasMany)
المشروع ←→ الملفات (hasMany)
المشروع ←→ المصروفات (hasMany)
المشروع ←→ الاجتماعات (hasMany)
المشروع ←→ تسجيلات الوقت (hasMany)
المشروع ←→ قناة محادثات (auto-created)

المهمة ←→ المشروع (project_id)
المهمة ←→ العميل (client_id)
المهمة ←→ المسؤول الرئيسي (assigned_to)
المهمة ←→ المسؤولون الإضافيون (task_assignees pivot)
المهمة ←→ المهام الفرعية (parent_id self-reference)
المهمة ←→ التعليقات (hasMany)
المهمة ←→ قوائم المراجعة (hasMany)
المهمة ←→ تسجيلات الوقت (hasMany)
المهمة ←→ العلامات Tags (morphMany)
```

## 🎯 User Journey - الموظف

### السيناريو: موظف يتابع مهامه اليومية

1. **يفتح الداشبورد** → يرى عدد مهامه، المهام المتأخرة، معدل الإنجاز
2. **يذهب للوحة Kanban** (`/tasks/board`) → يرى مهامه فقط (المنسوبة إليه)
3. **يسحب مهمة** من "جديد" إلى "جاري التنفيذ" → تتحدث الحالة تلقائيًا
4. **يضغط على المهمة** → تفتح صفحة التفاصيل
5. **يبدأ التايمر** → يبدأ تسجيل الوقت
6. **يعمل على المهمة** ويأشر على بنود قائمة المراجعة
7. **يضيف تعليق** مع ملف مرفق إذا لزم الأمر
8. **يوقف التايمر** ← يُحفظ الوقت
9. **يغير الحالة** إلى "مراجعة" ← يُرسل إشعار للمدير
10. **المدير يراجع** ويغير الحالة إلى "مكتمل" ← يُرسل إشعار للموظف

### السيناريو: مدير يتابع فريقه

1. **يفتح الداشبورد** → يرى المشاريع النشطة، معدل الإنجاز، المهام المتأخرة
2. **يفتح نظرة عامة المهام** (`/tasks-hub`) → رسم بياني شامل
3. **يفتح قائمة المهام** (`/tasks`) → يرى كل مهام الشركة
4. **يفلتر بالمسؤول** لمتابعة موظف محدد
5. **ينشئ مهمة جديدة** ويسندها لموظف ← يُرسل إشعار للموظف
6. **يفتح مشروع** → يرى التقدم الكلي ومهام كل موظف
7. **يتابع تقارير الأداء** (`/reports`) → يرى ساعات العمل وإنجاز الفريق

## ⚙️ ملاحظات تقنية

- **Multi-tenant**: كل البيانات مرتبطة بـ `company_id` عبر `HasCompany` trait
- **Slug-based routing**: المشاريع تُعرض بالـ slug وليس الـ ID
- **Real-time timer**: يتم استطلاع التايمر من السيرفر كل 30 ثانية
- **File storage**: الملفات تُحفظ في `storage/app/public` بمسار `projects/{id}/`
- **Workflow integration**: إكمال المهمة يُفعّل قواعد الأتمتة
- **Notification system**: إشعارات متكاملة عبر `NotificationService`
